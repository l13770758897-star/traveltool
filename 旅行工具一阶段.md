# 旅行工具 - 一阶段文档

> 极简旅行攻略编辑器 | 基于高德地图的行程规划工具

---

## 一、项目概述

### 1.1 项目定位
一款轻量级、纯前端（无后端）的旅行行程规划工具，用户可以在地图上搜索目的地、添加行程点位、自动规划驾车路线、调整顺序、预览并导出行程。

### 1.2 技术栈
| 类别 | 技术 |
|---|---|
| 地图服务 | 高德地图 JS API v2.0 |
| 数据存储 | localStorage + IndexedDB + File System API |
| 语言 | 原生 JavaScript (ES6+) |
| 样式 | 内联 CSS |
| 构建 | 无构建工具，直接打开 `index.html` 即可运行 |

### 1.3 项目结构
```
traveltool/
├── index.html          # 主页面（UI 结构 + 内联样式）
├── script.js           # 核心业务逻辑（地图操作、数据管理、预览渲染）
├── file-handle.js      # 文件系统操作（File System API + IndexedDB）
├── dev.sh              # 本地开发启动脚本
├── .gitignore          # Git 忽略规则
└── travel_data.json    # 示例数据文件（可选）
```

## 二、一阶段已实现功能

### 2.1 地图与搜索
- **地图初始化**：加载高德地图，默认中心为北京，缩放级别 11，浅色主题
- **目的地搜索**：输入框 + 高德 PlaceSearch，搜索结果以列表和地图标记展示
- **搜索交互**：点击搜索结果定位并缩放地图，鼠标悬停标记显示详情弹窗
- **一键添加**：搜索结果弹窗中可直接将目的地添加到行程

### 2.2 点位管理
- **添加点位**：点击地图或搜索结果添加点位，填写名称、描述、逗留时间、日期、类型
- **编辑点位**：点击地图上已有点位，弹出编辑表单，修改后实时更新
- **删除点位**：通过编辑表单或点位列表删除
- **点位类型**：5 种预设类型（默认、酒店、景点、美食、拍照点），每种有独立颜色标识
- **天气信息**：添加点位时自动通过逆地理编码获取城市，调用高德天气 API 获取实时天气

### 2.3 路线规划
- **自动连线**：添加/删除/编辑点位后自动调用高德 Driving API 规划驾车路线
- **途经点支持**：支持起点 → 多个途经点 → 终点的多段路线规划
- **路线面板**：右侧面板（`#routeList`）显示高德返回的详细驾车指引

### 2.4 顺序管理
- **拖拽排序**：点位列表支持拖拽调整顺序，拖拽后自动重新规划路线
- **序号标识**：地图标记根据顺序显示序号（起点和终点显示名称，途经点显示圆形序号）
- **标签刷新**：顺序变更后自动刷新所有标记的显示内容

### 2.5 预览与导出
- **预览模式**：全屏 Canvas 预览，支持两种模式：
  - **直线模式**：红色虚线连接各点位
  - **路线模式**：蓝色半透明实线显示实际驾车路线（使用 `routePath` 数据）
- **对话框展示**：每个点位旁显示白色圆角对话框，包含名称、描述、逗留时间、天气、类型、日期
- **防重叠算法**：迭代检测并推开重叠的对话框，确保任意密度下不重合
- **下载功能**：将 Canvas 预览导出为 PNG 图片

### 2.6 数据管理
- **多攻略支持**：支持创建多个独立攻略（itinerary），通过下拉列表切换
- **自动保存**：每次增删改操作自动保存到 localStorage
- **文件关联**：通过 File System API 关联本地 JSON 文件，实现自动持久化
  - IndexedDB 存储文件句柄，刷新页面后自动恢复权限
  - 支持新建文件、关联已有文件、切换文件
- **导入/导出**：支持 JSON 文件的导入和导出，方便跨设备迁移数据

## 四、代码架构说明

### 4.1 全局状态
```javascript
let map;                    // AMap.Map 实例
let markers = [];           // 当前攻略的 AMap.Marker 数组
let currentName = '';       // 当前攻略名称
let searchMarkers = [];     // 搜索结果标记（临时）
let markerIdCounter = 0;    // 标记 ID 计数器
let dataStore = {...};      // 完整数据（含所有攻略）
let routePath = null;       // 驾车路线坐标数组（用于预览路线模式）
let previewMode = 'line';   // 预览模式：'line' | 'road'
let driving;                // AMap.Driving 实例（路线规划）
```

### 4.2 核心函数清单
| 函数 | 职责 |
|---|---|
| `updateRoute()` | 清空旧路线，调用 Driving API 规划新路线，刷新标记标签和点位列表 |
| `refreshMarkerLabels()` | 根据点位顺序更新标记显示内容（起点/终点显示名称，途经点显示序号） |
| `renderPointList()` | 渲染右侧点位列表，绑定拖拽排序事件 |
| `renderPreview(mode)` | Canvas 渲染预览图（直线/路线模式 + 对话框） |
| `autoSave()` | 将当前 markers 同步到 dataStore，写入 localStorage 和关联文件 |
| `loadItinerary(name)` | 加载指定攻略，创建标记，恢复路线 |
| `loadDataStore()` | 页面加载时从文件或 localStorage 恢复数据 |

### 4.3 关键设计决策
1. **标记内容动态刷新**：通过 `refreshMarkerLabels()` 在每次路线更新后统一刷新，避免内容不一致
2. **预览对话框防重叠**：两阶段布局（初始定位 → 迭代推开重叠），最多 30 次迭代
3. **文件句柄持久化**：使用 IndexedDB 存储 FileSystemHandle，避免每次刷新都重新选择文件
4. **降级策略**：优先从关联文件加载 → 降级 localStorage → 兼容旧版 itinerary_ 前缀数据

## 五、已知限制与注意事项

### 5.1 浏览器兼容性
- **File System API**：仅 Chrome/Edge 86+ 支持，Firefox/Safari 不可用
- **降级方案**：不支持 File System API 的浏览器自动回退到 localStorage

### 5.2 高德 API 限制
- **Driving API 途经点数量**：最多支持 16 个途经点（含起点终点共 18 个点）
- **天气 API**：免费版有调用频率限制
- **安全密钥**：当前硬编码在 HTML 中，生产环境建议通过环境变量注入

### 5.3 数据存储
- **localStorage 容量**：约 5-10MB，大量攻略或图片数据可能超出限制
- **文件关联**：关联的本地文件被外部修改后不会自动同步，需重新导入

### 5.4 预览功能
- **Canvas 尺寸**：固定 1200×800，大量点位时对话框可能超出画布
- **路线模式依赖**：`routePath` 数据来自 Driving API 异步回调，首次打开预览可能路线未加载完成

## 六、二阶段开发建议

### 6.1 功能扩展方向
| 优先级 | 功能 | 说明 |
|---|---|---|
| P0 | 行驶时间计算与显示 | 在地图和列表上显示每段驾车时间（需解决 AMap 并发请求问题） |
| P0 | 多日行程拆分 | 支持按日期自动分组点位，生成每日行程视图 |
| P1 | 路线优化 | 根据距离/时间自动推荐最优访问顺序（TSP 问题） |
| P1 | 离线地图缓存 | 缓存瓦片数据，支持离线查看 |
| P2 | 多人协作 | 通过云端同步实现多人编辑同一攻略 |
| P2 | 预算估算 | 根据里程、油价、住宿类型估算旅行成本 |
| P3 | 导航导出 | 导出为 GPX/KML 格式，导入到车载导航或手机地图 |
| P3 | 天气预测 | 根据行程日期查询历史天气或天气预报 |

### 6.2 技术改进方向
| 方向 | 建议 |
|---|---|
| 代码组织 | 拆分为模块化文件（map.js, data.js, preview.js, ui.js） |
| 类型安全 | 引入 TypeScript 或 JSDoc 类型注解 |
| 构建工具 | 引入 Vite/Webpack 支持热更新和代码压缩 |
| 测试 | 为核心函数（路线规划、数据序列化）编写单元测试 |
| 性能 | 大量点位时 Canvas 渲染优化（虚拟滚动、分层渲染） |
| 安全 | 将 API Key 移至环境变量，避免硬编码 |

## 七、快速开始

### 7.1 本地运行
```bash
# 方式 1：直接打开
open index.html

# 方式 2：使用 dev.sh 启动本地服务器
./dev.sh
# 然后访问 http://localhost:8000
```

### 7.2 关联数据文件（推荐）
1. 点击右上角「新建」按钮创建 JSON 数据文件
2. 或点击「⚠️ 未关联文件」选择已有 JSON 文件
3. 关联后所有操作自动保存到该文件

### 7.3 导入已有数据
1. 点击「导入」按钮
2. 选择之前导出的 JSON 文件
3. 数据自动加载到当前会话
